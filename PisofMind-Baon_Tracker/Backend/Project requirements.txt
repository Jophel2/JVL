Below is a **Low-Level Requirements Document (LLRD) for PisofMind**, structured for a **working MVP first**, using **Java (Spring-style backend) + React-Typescript frontend**, with **simple, stable architecture and proper prioritization**.

---

# 📘 Low-Level Requirements Document (LLRD)

## Project: PisofMind (MVP Version)

---

# 1. 🎯 MVP Goal (Critical Clarification)

The MVP must allow students to:

1. Register and login
2. Set a budget
3. Add expenses under categories
4. View expense history
5. View summary of spending
6. Earn simple gamification rank based on savings behavior

👉 Anything beyond this (advanced ranking, complex analytics) is **Phase 2**

---

# 2. 🧱 System Architecture (Simple + Stable)

## Backend (Java - Spring Boot style)

* Layered architecture:

  * Controller Layer (API)
  * Service Layer (Business Logic)
  * Repository Layer (Data Access)
  * Entity Layer (Domain Model)

## Frontend (React + TypeScript)

* Component-based architecture
* Simple state handling (useState/useEffect or Context API only)
* No Redux in MVP

---

# 3. 👤 Core Entities (Domain Model - Java OOP)

```java
class User {
    Long id;
    String email;
    String password;
    String gender;
    int age;

    BigDecimal budget;

    List<Expense> expenses;
    UserRank rank;
```

## 3.2 Expense

```java
class Expense {
```

# 4. 🔐 Authentication Module (MVP Priority 1)

## 4.1 API Endpoints

### Register User

```
POST /api/auth/register
```

Request:

```json
{
  "email": "",
  "password": "",
  "gender": "",
  "age": 0
}
```

Response:

```json
{
  "message": "User registered successfully"
}
```

---

### Login User

```
POST /api/auth/login
```

Request:

```json
{
  "email": "",
  "password": ""
}
```

Response:

```json
{
  "token": "jwt-token",
  "userId": 1
}
```

---

## 4.2 Service

### AuthService

* validate registration
* hash password
* authenticate user
* generate JWT token

---

## 4.3 Design Pattern

* **Service Layer Pattern**
* **DTO Pattern**
* **JWT Authentication**

# 8. 🏆 Gamification System (MVP SIMPLE VERSION)

## 8.1 Rule (KEEP SIMPLE FOR MVP)

Rank increases based on:

* Staying within budget
* Consistent logging of expenses

---

## 8.2 Service

```java
class GamificationService {

    UserRank calculateRank(User user) {
        if (user.totalSavingsDuration >= 12 months)
            return IMMORTAL_SAVER;

        if (9 months) return MYTHICAL_SAVER;
        if (6 months) return LEGENDARY_SAVER;
        if (3 months) return MASTER_SAVER;

        return ROOKIE_SAVER;
    }
}
```

---

## 8.3 MVP Simplification

❌ No leaderboard yet
❌ No real-time ranking
✔ Only personal rank calculation
